# 📈 Stock-Analysis-System

このシステムは、**企業の決算情報や開示ニュースを24時間自動で収集し、AIで分析してSlackに通知・Googleスプレッドシートに記録する自動監視システム**です。

PDFの決算短信もOCR（文字認識）で読み取り、機関投資家のような視点で「強気」「弱気」を自動判定します。

---

### 📂 フォルダ階層図

```text
stock-analysis-system/          ← (システム本体)
│
├── .github/                    ← GitHub専用の設定フォルダ
│   └── workflows/              ← 自動化の「ワークフロー」を入れる場所
│       └── run_schedule.yml    ← ★「30分に1回動け」という指令書 (OCR等のインストール指示含む)
│
├── main.py                     ← ★プログラムの本体 (ロジックの中枢)
│
├── sources.json                ← 収集するRSSのURL一覧
│
├── watch_list.txt              ← ★監視したい銘柄コードリスト (Yanoshin用フィルタ)
│
└── requirements.txt            ← 必要なPythonライブラリの一覧
```

---

### 各ファイルの詳しい役割

#### 1. `main.py`
**システムの本体**です。

*   **何をしているか？**
    *   RSSからニュースを集める（Yanoshinとそれ以外で挙動を切り替え）。
    *   `watch_list.txt` に基づき、不要な銘柄へのアクセスをカットする。
    *   PDFや画像をOCR処理してテキスト化する。
    *   「上方修正」などのキーワードで選別し、GPT-5.2に分析させる。
    *   Slackに通知し、Google Sheetsに保存する。

#### 2. `watch_list.txt`
**監視したい企業の銘柄コード（4桁）を指定するリスト**です。

*   **役割:**
    *   ここにコード（例: `1301`）を書くと、`sources.json` に大量のURLがあっても、**指定した銘柄のYanoshin RSSだけ**をチェックします。
    *   **空の場合:** 制限なく、`sources.json` にある全てのRSSをチェックします。
*   **変更する場所:**
    *   監視対象の企業を絞り込みたい時に編集します。

#### 3. `sources.json`
データの元となるRSSのURLリストです。

*   **中身:**
    *   `webapi.yanoshin.jp`（適時開示）や、PR TIMES、JPXなどのRSS URL。
*   **役割:**
    *   システムが巡回するWebサイトのリストです。
*   **変更する場所:**
    *   新しい情報源サイトを追加したい時に編集します。

#### 4. `requirements.txt`
Pythonライブラリのリストです。

*   **役割:**
    *   `openai`, `feedparser`, `pdf2image`, `pytesseract` などが含まれます。GitHub Actions起動時に自動インストールされます。

#### 5. `.github/workflows/run_schedule.yml`
24時間自動稼働のための**指令書**です。

*   **役割:**
    *   30分ごとの定期実行 (`cron: '*/30 * * * *'`)。
    *   **重要:** Pythonだけでなく、**OCRエンジン（Tesseract）やPDFツール（Poppler）をOSにインストールする指示**もここに書かれています。

## ⚙️ システムの挙動ロジック

本システムは、サイトの種類によって収集モードを自動で切り替えます。

### 1. Yanoshin（適時開示 RSS）の場合
*   **監視リスト機能**: `watch_list.txt` に記載がない銘柄はアクセスせずにスキップします（高速化）。
*   **新着＆過去収集モード**:
    *   通常は「直近3日間」の新着のみを監視します。
    *   **もし新着で重要キーワード（決算など）が見つかった場合**、自動的に「過去データ収集モード」に切り替わり、そのRSSに含まれる**過去5年分のデータ**もまとめて収集・分析します。

### 2. その他のサイト（PR TIMES, JPXなど）の場合
*   常に「直近3日間」の新着記事のみを収集します。
*   過去のノイズデータを拾わないよう、厳密に時間でフィルタリングされます。

## 🚀 導入手順 (セットアップ)

### ステップ1：APIキーの設定
GitHubのリポジトリ設定（Settings → Secrets and variables → Actions）に、以下の3つを登録してください。

| Name（名前） | Secret（値・中身） |
| :--- | :--- |
| **OPENAI_API_KEY** | OpenAI APIキー (`sk-`から始まる文字列) |
| **SLACK_WEBHOOK_URL** | 通知先のSlack Webhook URL (`https://hooks.slack...`) |
| **GOOGLE_CREDENTIALS_JSON** | Google CloudのJSONファイルの中身すべて<br>※スプレッドシート連携用 |

### ステップ2：スプレッドシートの共有
Google Cloudで発行したサービスアカウントのメールアドレス（`xxxx@...iam.gserviceaccount.com`）に対し、ログ保存用スプレッドシート（`stock_analysis_log`）の**「編集者」権限**を共有してください。

### ステップ3：監視リストの作成
リポジトリ直下に `watch_list.txt` を作成し、監視したい銘柄コードを改行区切りで入力します。
（全銘柄を監視したい場合は、ファイルを空にするか削除してください）

```text
1301
6758
9984
```

## 🔧 カスタマイズ方法

### 1. 検索ワード・フィルタの調整
`main.py` 内の設定変数を変更します。

*   **`TARGET_KEYWORDS`**
    *   RSSの記事タイトルに含まれるべき必須キーワード（例: "決算", "修正"）。
*   **`POSITIVE_WORDS` / `NEGATIVE_WORDS`**
    *   AI分析の前段階で、記事本文に含まれているかチェックする単語リスト。

### 2. AIへの指示（プロンプト）の調整
`main.py` の `analyze_gpt` 関数内にある `prompt` 変数を編集します。
「もっと辛口に評価して」「要約は1行で」などの指示変更が可能です。

## ⚠️ 注意事項

1.  **システム依存ツール**:
    このシステムはOCR機能を使用するため、ローカル環境で動かす場合は `tesseract-ocr` と `poppler-utils` のインストールが必要です（GitHub Actions上では自動で行われます）。
    
2.  **GitHub Actionsの仕様**:
    *   **開始遅延**: 30分ごとの設定でも、サーバー混雑時は数分遅れることがあります。システム側で「3日間のバッファ」を持たせているため、遅れても取りこぼしはありません。
    *   **60日ルール**: リポジトリ更新が60日間ないと自動停止します。停止メールが来たら「Enable workflow」ボタンを押してください。

3.  **APIコスト**:
    OpenAI APIは従量課金です。決算シーズンなどで大量のPDFを分析するとコストが増加する可能性があります。

### 【マニュアル】OPENAI_API_KEY とは？

今回構築したシステムでは、文章を分析するためにOpenAI社の「GPT」を使用します。
`OPENAI_API_KEY` は、AIを利用するために必要な「**デジタル通行手形**」のようなものです。

#### 1. 何に使うもの？
GitHubにあるプログラムが、OpenAI社のAIに接続する際の証明をするために使います。これがないと、システムはAIを動かすことができず、分析ができません。

#### 2. 料金について（重要）
このキーを使うと、AIが文章を読んだり書いたりする量に応じて「**従量課金（使った分だけ支払い）**」が発生します。

*   **月額20ドルの「ChatGPT Plus」とは別物です。**
    *   ChatGPT Plus（チャット画面）は、人間が使うための定額制プランです。
    *   API（今回のシステム）は、プログラムが使うための従量制プランです。別途クレジットカードの登録が必要です。
*   **費用感の目安:**
    *   今回のシステムで、決算書1通をGPT-5.2で分析する場合、おおよそ数銭〜数円程度のコストがかかります（文章量によります）。
    *   使わなければ0円です。

#### 3. 取得方法
まだお持ちでない場合は、以下のOpenAI公式ページから発行できます。
1.  [OpenAI Platform](https://platform.openai.com/) にアクセス・ログインする。
2.  「Settings」→「Billing」でクレジットカードを登録する（※ここが必須です）。
3.  「API Keys」のメニューから「Create new secret key」を押して発行する。
    *   `sk-...` から始まる長い文字列が発行されます。これがキーです。

### 【マニュアル】SLACK_WEBHOOK_URL とは？

`SLACK_WEBHOOK_URL` は、今回作成したAIシステムから、Slackへメッセージを届けるための「**専用の郵便ポストの住所**」のようなものです。

#### 1. 何に使うもの？
このシステムはクラウド（GitHub）上で動いていますが、分析が終わった瞬間に、結果をスマートフォンやPCのSlackに通知する必要があります。
「このURLにデータを送ると、Slackの『◯◯チャンネル』にメッセージが表示される」という仕組みになっています。

#### 2. どんな見た目？
`https://hooks.slack.com/services/...` から始まる、非常に長い英数字の文字列です。
これが「システムとSlackをつなぐ直通電話番号」の役割を果たします。

#### 3. 取得方法（発行の手順）
このURLは、ご自身のSlack管理画面から無料で発行できます。以下の手順で行ってください。

1.  PCのブラウザで [Slack API: Your Apps](https://api.slack.com/apps) にアクセスします。
2.  **「Create New App」** ボタンを押し、**「From scratch」** を選びます。
    *   **App Name:** 何でもOKです（例: `Stock-Bot`）
    *   **workspace:** 通知を送りたいSlackワークスペースを選択
    *   「Create App」ボタンを押します。
3.  左側のメニューから **「Incoming Webhooks」** をクリックします。
4.  右上のスイッチを **「On」** に切り替えます。
5.  画面下部に出現する **「Add New Webhook to Workspace」** ボタンを押します。
6.  通知を送りたい「チャンネル（例: #general や #決算速報）」を選び、「許可する」を押します。
7.  画面に **「Webhook URL」** が表示されますので、「Copy」ボタンを押してコピーしてください。

このコピーしたURLを、GitHubの設定画面（Secrets）に登録します。

*   「`xxxx@Stock-Analysis.iam.gserviceaccount.com` のような長いメールアドレスです。これをコピーしてください。」
*   「今回使うGoogleスプレッドシート（`stock_analysis_log`）を開いてください。」
*   「右上の『共有』ボタンを押し、先ほどのアドレスを貼り付けて、『送信』を押してください。」

### 【マニュアル】システムの停止・再開方法

年末年始や、通知を一時的に止めたい場合などは、以下の手順でシステムを停止（休止）させることができます。

#### 1. システムを停止する手順
1.  GitHubのリポジトリページ（トップ画面）を開きます。
2.  画面上部のタブメニューから **「Actions」** をクリックします。
3.  左側のメニューにある **「Stock Analysis 24/7」** をクリックします。
4.  画面の右上にある **「...（3点リーダー）」** ボタンをクリックします。
5.  出てきたメニューの中から **「Disable workflow」** をクリックします。

これでシステムは完全に停止し、自動巡回も行われなくなります。
※停止している間は、API利用料もサーバー負荷も一切かかりません。

#### 2. システムを再開する手順
同じ画面で **「Enable workflow」** という緑色のボタンが表示されていますので、それをクリックするだけで、即座に24時間監視が再開されます。

## 🔧 後から追加・変更する要素マニュアル

運用を始めると、「もっとこういうニュースが欲しい」「この通知はいらない」といった要望が出てきます。その際の調整ガイドです。

### 1. 検索ワード・フィルタの調整（一番よく触る場所）
「ニュースを拾いすぎる（ノイズが多い）」または「重要な記事が漏れている」場合に修正します。

*   **ファイル:** `main.py`
*   **箇所:** 40行目付近 `TARGET_KEYWORDS`
    *   **役割:** RSSの記事タイトルにこの言葉が含まれていないと無視します（第一関門）。
    *   **変更例:**
        *   **漏れがある:** `"業務提携"`, `"月次"`, `"自社株買い"` を追加する。
        *   **ノイズが多い:** 一般的すぎる言葉（もしあれば）を削除する。

*   **箇所:** 26〜37行目付近 `POSITIVE_WORDS` / `NEGATIVE_WORDS`
    *   **役割:** 記事の中身（本文）にこれらの単語があるかチェックします（第二関門）。
    *   **変更例:**
        *   **特定のテーマを追いたい:** `"防衛"`, `"生成AI"`, `"値上げ"` などをリストに追加すると、それらが含まれた際にAI分析へ回りやすくなります。

### 2. 監視対象の変更（企業・サイト）
監視したい企業を変えたり、情報源を増やしたい場合に修正します。

*   **ファイル:** `watch_list.txt` （Yanoshin用）
    *   **役割:** 適時開示（Yanoshin）で監視する企業のコードリスト。
    *   **変更例:**
        *   新しい監視銘柄ができたら、その「4桁コード」を改行して追記する。
        *   全銘柄監視に戻したいときは、中身を全て消す（空にする）。

*   **ファイル:** `sources.json` （RSS URL）
    *   **役割:** 情報収集先のURLリスト。
    *   **変更例:**
        *   新しいニュースサイト（RSS対応）を見つけたら、そのURLをリストに追加する。
        *   **注意:** JSON形式（カンマ区切り、ダブルクォート）のルールを壊さないように注意してください。

### 3. AIへの指示（プロンプト）の調整
「AIの分析が甘い」「要約が長すぎる」「もっと株価への影響を断言してほしい」といった場合に修正します。

*   **ファイル:** `main.py`
*   **箇所:** `analyze_gpt` 関数内の `prompt` 変数（190行目付近）
    *   **役割:** AIへの指令書です。
    *   **変更例:**
        *   **辛口にしたい:** 「投資家として、リスク要因を洗い出し、悲観的なシナリオも想定して評価してください」と書き足す。
        *   **要約を短くしたい:** 「内容の要約（3行以内）」を「内容の要約（50文字以内）」に変更する。
        *   **モデル変更:** `GPT_MODEL = "gpt-5.2"` の部分を、将来新しいモデル（例: `"gpt-6"`）が出たら書き換える。

### 4. 通知のデザイン・宛先
Slackの見た目や、メンション（通知音）を変えたい場合です。

*   **ファイル:** `main.py`
*   **箇所:** `notify_slack` 関数内（230行目付近）
    *   **役割:** Slackに送るメッセージの組み立て。
    *   **変更例:**
        *   **全員に通知:** `text` の部分に `<!channel>` を追加すると、Slack側で全員にメンションが飛びます（緊急時用）。
        *   **色分け:** 現在は「強気＝緑」「弱気＝赤」ですが、条件を変えたり色コードを変更できます。

### 5. 実行頻度の変更
「30分に1回じゃ遅い」「1時間に1回で十分」という場合です。

*   **ファイル:** `.github/workflows/run_schedule.yml`
*   **箇所:** `cron: '*/30 * * * *'`
    *   **変更例:**
        *   **15分ごとに変更:** `'*/15 * * * *'`
        *   **1時間に1回に変更:** `'0 * * * *'`
    *   **注意:** 頻度を上げすぎると（5分毎など）、OpenAIのAPI利用料が増えたり、サーバー側（Yanoshinなど）に迷惑がかかる可能性があるため、15分〜30分程度が推奨です。

### ⚠️ 変更時の注意点（ここだけは守ってください）

1.  **バックアップをとる**
    *   コードをいじる前に、動いている状態のコードをコピーしてどこかに保存しておくと安心です。
2.  **全角文字に気をつける**
    *   PythonやJSONファイルの中で、**全角スペース**や**全角の引用符（””）**を使ってしまうと、即座にエラーで動かなくなります。
    *   必ず**半角英数モード**で記号を入力してください。
3.  **watch_list.txt は数字だけ**
    *   このファイルに「1301 極洋」のように社名まで書いてしまうと動きません。「1301」のように数字だけを書いてください。
